# لاCMS

سیستم مدیریت محتوا (CMS) مدرن و قدرتمند برای وب سایت‌های فارسی و انگلیسی که بر پایه Laravel و Livewire ساخته شده است.

## درباره لاCMS

**لاCMS** یک سیستم مدیریت محتوای تمام‌عیار است که امکانات جامعی برای:
- مدیریت پست‌های وبلاگ
- مدیریت صفحات ایستا
- تنظیم دسته‌بندی‌ها و برچسب‌ها
- مدیریت نظرات کاربران
- اپلود و مدیریت تصاویر
- احراز هویت و مدیریت کاربران

فراهم می‌کند. واسط کاربری تعاملی و بدون نیاز به صفحه‌سازی سنتی (SPA) با استفاده از فناوری Livewire ساخته شده است.

## پیش‌نیازها

- **PHP**: 8.4+
- **Composer**: برای مدیریت وابستگی‌های PHP
- **Node.js & npm**: برای مدیریت وابستگی‌های فرانت‌اند
- **Git**: برای نسخه‌گذاری

## فناوری‌های استفاده شده

### بک‌اند
- **Laravel 13**: فریم‌ورک وب قدرتمند و الگان
- **Livewire 4**: ساخت واسط‌های تعاملی بدون JavaScript خالص
- **PHP 8.4**: زبان برنامه‌نویسی

### فرانت‌اند
- **TailwindCSS 4**: فریم‌ورک CSS کاربردی
- **Vite**: ابزار بندی سریع و مدرن
- **DaisyUI**: کتابخانه اجزاء UI بر اساس TailwindCSS
- **Editor.js**: ویرایشگر محتوای بدون WYSIWYG

### ابزارها
- **Pest**: فریم‌ورک تست مدرن PHP
- **Laravel Pint**: ابزار فرمت‌کردن کد PHP
- **Laravel Boost**: ابزارها و مهارت‌های کمکی برای AI

## ساختار پروژه

```
lacms/
├── app/
│   ├── Models/              # مدل‌های Eloquent (Post, Category, Page, Tag, etc)
│   ├── Http/
│   │   └── Controllers/     # کنترلرهای HTTP
│   └── Livewire/           # اجزاء Livewire برای فرانت‌اند
├── database/
│   ├── migrations/          # مهاجرت‌های پایگاه داده
│   ├── factories/           # فکتوری‌های تست
│   └── seeders/            # بذرپاشی داده‌های پایگاه داده
├── routes/
│   ├── web.php             # مسیرهای عمومی و ادمین
│   └── auth.php            # مسیرهای احراز هویت
├── resources/
│   └── views/              # Blade templates
├── tests/
│   ├── Feature/            # تست‌های ویژگی
│   └── Unit/               # تست‌های واحدی
├── public/                 # فایل‌های عمومی (images, etc)
└── config/                 # فایل‌های تنظیمات
```

## روند کار برنامه

1. **مسیریابی**: بازدیدکنندگان به صفحات عمومی و وبلاگ دسترسی دارند
2. **احراز هویت**: کاربران می‌توانند ثبت نام و وارد شوند
3. **مدیریت محتوا**: ادمین‌ها در داشبورد می‌توانند پست‌ها، صفحات و دسته‌بندی‌ها را مدیریت کنند
4. **Livewire**: اجزاء Livewire به‌طور بدون‌پایه‌نویسی با سرور ارتباط برقرار می‌کنند
5. **پایگاه داده**: تمام داده‌ها در SQLite یا دیگر پایگاه‌های داده ذخیره می‌شوند

## نصب و راه‌اندازی

### 1. کلون کردن مخزن
```bash
git clone https://github.com/asgarimehdi/lacms.git
cd lacms
```

### 2. راه‌اندازی اولیه (خودکار)
```bash
composer run setup
```

این دستور به‌طور خودکار:
- وابستگی‌های Composer را نصب می‌کند
- فایل `.env` را ایجاد می‌کند
- کلید برنامه را تولید می‌کند
- مهاجرت‌های پایگاه داده را اجرا می‌کند
- وابستگی‌های npm را نصب می‌کند
- دارایی‌های فرانت‌اند را می‌سازد

### 3. اجرای توسعه

#### گزینه‌ی اول: با مدیر فرایند (توصیه شده)
```bash
composer run dev
```

این دستور به‌طور هم‌زمان اجرا می‌کند:
- سرور توسعه Laravel (پورت 8000)
- تعطیل صف کاری (queue listener)
- خط لوله Vite برای دارایی‌های فرانت‌اند

#### گزینه‌ی دوم: اجرا به‌طور جداگانه

**پایانه‌ی 1: سرور Laravel**
```bash
php artisan serve
```

**پایانه‌ی 2: صف کاری**
```bash
php artisan queue:listen --tries=1
```

**پایانه‌ی 3: Vite (توسعه)**
```bash
npm run dev
```

## استفاده

### ورود به داشبورد
پس از راه‌اندازی، داشبورد ادمین در دسترس است:
- **آدرس**: `http://localhost:8000/admin`
- **عملکردها**:
  - ✍️ ایجاد و ویرایش پست‌های وبلاگ
  - 📄 مدیریت صفحات ایستا
  - 📁 سازماندهی دسته‌بندی‌ها و برچسب‌ها
  - 👥 مدیریت کاربران و نظرات
  - ⚙️ تنظیمات سایت

### مشاهده‌ی صفحات عمومی
- **صفحه‌ی اصلی**: `http://localhost:8000/`
- **وبلاگ**: `http://localhost:8000/blog`
- **دسته‌بندی**: `http://localhost:8000/category/{slug}`
- **برچسب**: `http://localhost:8000/tag/{slug}`

## تست

اجرای کل تست‌های Pest:
```bash
php artisan test
```

اجرای تست‌های خاص:
```bash
php artisan test --compact --filter=LoginTest
```

اجرای تست‌های فشرده:
```bash

```

بر اساس استاندارد Laravel Pint:
```bash
vendor/bin/pint
```

تنها فایل‌های تغییر یافته:
```bash
vendor/bin/pint --dirty
```

## میزان‌های حفاظت (Rate Limiting)

برنامه حفاظت‌های زیر را اعمال می‌کند:
- **ورود و ثبت‌نام**: 5 تلاش در دقیقه
- **مشاهده‌ی پست**: 3 تلاش در دقیقه
- **اپلود تصویر**: 10 تلاش در دقیقه

## مدل‌های داده

### Post (پست)
```php
- id
- user_id        // نویسنده پست
- category_id    // دسته‌بندی
- title          // عنوان
- slug           // نشانی شناخت (URL-friendly)
- content        // محتوا
- is_published   // منتشر شده / پیش‌نویس
- timestamps
```

### Category (دسته‌بندی)
```php
- id
- name           // نام دسته‌بندی
- slug           // نشانی شناخت
- timestamps
```

### Page (صفحه)
```php
- id
- title          // عنوان صفحه
- slug           // نشانی شناخت
- content        // محتوا
- status         // وضعیت (draft, published)
- parent_id      // صفحه‌ی والد (برای ساختار سلسله‌مراتبی)
- sort           // ترتیب نمایش
- timestamps
```

### User (کاربر)
```php
- id
- name
- email
- password
- timestamps
```
